fn activation_root(&self) -> Option<File>[src]

Expand description

Gets the activation root for a crate::Volume if it is known ahead of mount time. Returns None otherwise. If not None and if self is mounted, then the result of [crate::Mount::get_root()] on the crate::Mount object obtained from [crate::Volume::get_mount()] will always either be equal or a prefix of what this function returns. In other words, in code

  GMount *mount;
  GFile *mount_root
  GFile *volume_activation_root;

  mount = g_volume_get_mount (volume); // mounted, so never NULL
  mount_root = g_mount_get_root (mount);
  volume_activation_root = g_volume_get_activation_root (volume); // assume not NULL

then the expression

  (g_file_has_prefix (volume_activation_root, mount_root) ||
   g_file_equal (volume_activation_root, mount_root))

will always be true.

Activation roots are typically used in crate::VolumeMonitor implementations to find the underlying mount to shadow, see [crate::Mount::is_shadowed()] for more details.

Returns

the activation root of self or None. Use [crate::glib::object::ObjectExt::unref()] to free.
