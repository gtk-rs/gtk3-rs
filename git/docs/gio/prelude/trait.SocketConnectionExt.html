fn remote_address(&self) -> Result<SocketAddress, Error>[src]

Expand description

Try to get the remote address of a socket connection.

Since GLib 2.40, when used with crate::prelude::SocketClientExt::connect() or crate::prelude::SocketClientExt::connect_async(), during emission of crate::SocketClientEvent::Connecting, this function will return the remote address that will be used for the connection. This allows applications to print e.g. “Connecting to example.com (10.42.77.3)…”.

Returns

a crate::SocketAddress or None on error. Free the returned object with [crate::glib::object::ObjectExt::unref()].
